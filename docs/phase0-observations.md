# Phase 0 Observations

## 我們做出了什麼？

- 做出一個可切換的原始資訊與整理工作台畫面。
- 讓使用者可以為 6 筆資訊建立、編輯、重設與刪除整理草稿。
- 把人類質疑內容與「不可直接行動」的限制明確顯示出來。

## 哪些資料讓 agent 很容易整理？

- M-009 與 M-010 的現場更新比較像可被整理成候選狀態的資訊，因為它們有較明確的現場情境與更新時間。

## 哪些資料讓 agent 亂補或誤判？

- M-001、M-004、M-007 這類社群與轉述內容很容易被誤當成已確認的現場事實。
- M-011 與 M-012 需要特別小心，因為裡面涉及長者隱私與家屬資訊，不能直接轉成任務。

## 哪些資訊不能直接變成任務？

- M-001 的清泥需求缺少完整地點與受災者身份。
- M-011 涉及長者與住家位置，必須先取得同意與確認。
- M-012 來自家屬來電，還沒有足夠現場資訊可建立任務。

## 哪些資訊的操作者不是當事人？

- M-004、M-007、M-012 都是由第三方或轉述者提供資訊，不應直接視為當事人本人回報。

## 我們剛剛偷偷假設了什麼？

- 我們假設原文中的關鍵字足以判斷候選類型，但實際上這仍然需要人類確認。

## 如果要讓這個工具變可靠，下一步需要什麼？

- 需要把每筆資訊的來源、時間、地點與確認者欄位更清楚地標示出來。
- 需要更明確的「為何不能直接行動」與「誰應該確認」的流程。

## 我們需要更清楚資料格式的地方

- 原始資料缺少明確的「確認者」、「確認時間」與「是否同意公開」欄位。

## 我們需要人工確認的地方

- 任何涉及人力派遣、住家位置與長者隱私的資訊，都要先由現場或主管確認。
